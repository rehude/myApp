# 🧭 总体思路（先建立正确模型）

你现在缺的是：

> ❌ “临时状态”（内存里的 token）
> ✅ “持久状态 + 会话状态 + UI状态”

我们要引入三层状态：

```text
持久层（磁盘） → token / provider
运行时（Rust State） → 当前登录态
前端状态 → UI控制（页面跳转）
```

# 🧱 一、问题1：每次重启都要输入 Token

## 🎯 目标

启动时自动恢复登录状态

---

## ✅ 方案：本地持久化（推荐用 tauri-plugin-store）

---

## 1️⃣ 加依赖（Rust）

```toml
tauri-plugin-store = "0.1"
```

---

## 2️⃣ 初始化（main.rs）

```rust
tauri::Builder::default()
  .plugin(tauri_plugin_store::Builder::default().build())
```

---

## 3️⃣ 存储结构（统一）

```json
{
  "provider": "github",
  "github_token": "...",
  "gitlab_token": "..."
}
```

---

## 4️⃣ 保存 Token（登录时）

```rust
pub async fn persist_auth(store: Store, state: &AppState) {
    store.set("provider", state.provider);
    store.set("github_token", state.github_token);
    store.set("gitlab_token", state.gitlab_token);
    store.save();
}
```

---

## 5️⃣ 启动恢复（关键）

在 `setup` 阶段：

```rust
fn load_auth(store: Store) -> AppState {
    AppState {
        provider: store.get("provider").unwrap_or(Provider::GitHub),
        github_token: store.get("github_token"),
        gitlab_token: store.get("gitlab_token"),
    }
}
```

---

## ✅ 前端启动逻辑（必须）

```ts
useEffect(() => {
  invoke("init_app").then((res) => {
    if (res.logged_in) {
      navigate("/repos");
    } else {
      navigate("/login");
    }
  });
}, []);
```

# 🔁 二、问题2：无法切换登录

## 🎯 目标

* 支持切换 GitHub / GitLab
* 支持切换账号（重新输入 token）

---

## ✅ 方案：引入“会话管理”

---

## 1️⃣ 新增 command

```rust
set_provider(provider)
set_github_token(token)
set_gitlab_token(token)
logout()
```

---

## 2️⃣ logout 实现（关键）

```rust
#[tauri::command]
async fn logout(state: State<AppState>, store: Store) {
    state.github_token = None;
    state.gitlab_token = None;

    store.delete("github_token");
    store.delete("gitlab_token");
    store.save();
}
```

---

## 3️⃣ 切换 provider（不清 token）

```rust
state.provider = Provider::GitLab;
```

👉 这样可以实现：

✔ GitHub / GitLab 快速切换
✔ 各自 token 保留

---

## 4️⃣ 前端 UI（必须有）

在 Header：

```text
[GitHub ▼] [Logout]
```

切换逻辑：

```ts
await invoke("set_provider", { provider: "gitlab" });
refreshRepos();
```

# 🧠 关键设计：统一错误模型

---

## ✅ Rust 返回结构（必须统一）

```rust
pub struct ApiResponse<T> {
    pub success: bool,
    pub data: Option<T>,
    pub error: Option<String>,
    pub code: Option<String>, // 新增
}
```

---

## 错误码定义（关键）

```text
UNAUTHORIZED
NETWORK_ERROR
UNKNOWN
```

---

## 示例：

```rust
ApiResponse {
  success: false,
  error: Some("Token invalid"),
  code: Some("UNAUTHORIZED"),
}
```

---

# 🔌 前端统一错误处理（必须做）

---

## 封装 invoke

```ts
async function callApi(cmd, args) {
  const res = await invoke(cmd, args);

  if (!res.success) {
    handleError(res);
    throw new Error(res.error);
  }

  return res.data;
}
```

---

## handleError（核心）

```ts
function handleError(res) {
  if (res.code === "UNAUTHORIZED") {
    // 清状态
    store.clear();

    // 跳登录
    navigate("/login");
  } else {
    alert(res.error || "Unknown error");
  }
}
```

---

# 🧩 Rust 错误映射（必须）

---

## 401 处理

```rust
if res.status() == 401 {
    return ApiResponse {
        success: false,
        code: Some("UNAUTHORIZED".into()),
        error: Some("Invalid token".into()),
        data: None,
    };
}
```

---

## 网络错误

```rust
Err(_) => ApiResponse {
    code: Some("NETWORK_ERROR".into()),
}
```

# 🔄 四、页面级兜底（防止白屏）

---

## React Router 加保护

```ts
function RequireAuth({ children }) {
  const token = useStore((s) => s.token);

  if (!token) {
    return <Navigate to="/login" />;
  }

  return children;
}
```
