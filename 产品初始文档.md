# 🔌 四、API 设计（关键，直接给前端用）

所有接口返回统一结构：

```ts
type ApiResponse<T> = {
  success: boolean;
  data?: T;
  error?: string;
};
```

------

## ✅ 1. 设置 Token

```ts
invoke("set_token", { token: string })
```

Rust：

```rust
#[tauri::command]
async fn set_token(state: State<AppState>, token: String) -> ApiResponse<()> 
```

逻辑：

- 存入 state
- 不做校验（避免复杂化）

------

## ✅ 2. 获取仓库列表

```ts
invoke("get_repos")
```

GitHub API：

```
GET https://api.github.com/user/repos
```

必须 Header：

```text
Authorization: Bearer TOKEN
User-Agent: tauri-app
Accept: application/vnd.github+json
```

------

返回结构（统一格式）：

```ts
type Repo = {
  id: number;
  name: string;
  full_name: string;
  private: boolean;
  html_url: string;
};
```

------

## ✅ 3. 仓库详情

```ts
invoke("get_repo_detail", { owner, repo })
```

API：

```
GET /repos/{owner}/{repo}
```

------

## ✅ 4. 获取提交列表

```ts
invoke("get_commits", { owner, repo })
```

API：

```
GET /repos/{owner}/{repo}/commits
```

返回结构：

```ts
type Commit = {
  sha: string;
  message: string;
  author: string;
  date: string;
};
```

------

## ✅ 5. 获取 commit 详情（diff）

```ts
invoke("get_commit_detail", { owner, repo, sha })
```

API：

```
GET /repos/{owner}/{repo}/commits/{sha}
```

关键点：

👉 diff 在这里：

```json
files[].patch
```
